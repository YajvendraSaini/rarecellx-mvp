# Security Policy

## Supported Versions

The following versions of the RareCellX project are currently being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a top priority for RareCellX, especially given the handling of inference data. We appreciate your efforts to responsibly disclose any vulnerabilities.

### How to Report

If you discover a security vulnerability within RareCellX, please **do not** open a public issue. Instead, please report it via one of the following methods:

1. **Email:** Send a detailed report directly to our team at **founder@rarecellx.app**. 
2. **GitHub Security Advisory:** Use the "Security" tab in this repository to report a vulnerability privately.

### What to Include
Please provide as much information as possible to help us reproduce and resolve the issue:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact (e.g., data leak, denial of service, remote code execution).
- Any related logs or screenshots.

### Response Timeline
- You can expect an initial acknowledgment of your report within **24 hours**.
- We will keep you updated on our progress as we investigate and develop a patch.

### 🏆 Security Hall of Fame
As a token of our gratitude for keeping our healthcare AI secure, anyone who responsibly discloses a verified vulnerability will be added to our public **Security Hall of Fame** (unless you prefer to remain anonymous).

Thank you for helping keep RareCellX safe!
